# FairyGUI Generator Pipeline And Stale Gate (#526)

## Goal

Turn the Runtime HUD FairyGUI manifest bootstrap into a repeatable noEngine
generation and validation path.

This slice does not call FairyGUI Editor and does not publish `*_fui.bytes`.
FairyGUI source XML remains the source of UI layout truth, while the generated
C# manifest is a checked-in adapter contract that must stay reproducible.

## Ownership Boundaries

Generator-owned:

- `Assets/Scripts/MxFramework/Demo/FairyGUI/Generated/RuntimeAbilitySliceFairyGuiHudManifest.g.cs`
- Future `Assets/Scripts/MxFramework/**/FairyGUI/Generated/**/*.g.cs` files
  added by explicit generator specs.

Manual code:

- `Assets/Scripts/MxFramework/Demo/FairyGUI/RuntimeAbilitySliceFairyGuiHudBinder.cs`
- `Assets/Scripts/MxFramework/Demo/FairyGUI/RuntimeAbilitySliceFairyGuiHudComposition.cs`
- `Assets/Scripts/MxFramework/Demo/FairyGUI/RuntimeAbilitySliceFairyGuiHudController.cs`
- `Assets/Scripts/MxFramework/Demo/FairyGUI/RuntimeAbilitySliceFairyGuiHudIds.cs`

FairyGUI Editor-owned:

- `FGUIProject/**`
- `Assets/Bundles/FGUI/**`

The noEngine generator may read FairyGUI source XML and compare generated C#
output. It must not overwrite manual binders, ids, composition code, FairyGUI
project files or published package bytes.

## Generator Path

The generator API lives in:

```text
Assets/Scripts/MxFramework/UI.FairyGUI.Manifest/MxFairyGuiManifestGenerator.cs
```

It accepts `MxFairyGuiManifestGenerationSpec`, reads the package/component XML,
projects text and button controls into `MxFairyGuiNamedControl`, combines
manual command and bind-path metadata, validates the generated manifest through
`MxFairyGuiManifestValidator`, and emits deterministic C# source text.

Generated output includes an `// <auto-generated />` header and no timestamp so
agents and CI can compare it reliably.

## Commands

Check whether checked-in generated output is current:

```bash
dotnet run --project Tools/MxFramework.NoEngineTests/FairyGUI.Manifest.Tests/FairyGUI.Manifest.Tests.csproj -- --check-generated
```

Regenerate the Runtime HUD manifest:

```bash
dotnet run --project Tools/MxFramework.NoEngineTests/FairyGUI.Manifest.Tests/FairyGUI.Manifest.Tests.csproj -- --write-runtime-hud-manifest
```

Run the full noEngine manifest validation suite:

```bash
dotnet run --project Tools/MxFramework.NoEngineTests/FairyGUI.Manifest.Tests/FairyGUI.Manifest.Tests.csproj
```

## Stale Gate

The stale gate compares the checked-in generated file with the in-memory output
from the current source XML and spec. A mismatch returns a structured
`GeneratedDescriptorMismatch` diagnostic and fails validation.

The gate intentionally does not compare FairyGUI package byte hashes. Package
bytes are validated separately by existence, `FGUI` header and resource catalog
mapping, because publish output is owned by FairyGUI Editor and may vary by
editor version.
