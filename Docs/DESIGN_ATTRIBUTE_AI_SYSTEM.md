## 一、设计背景与目标

### 项目现状

根据当前 VVV 仓库路线图，核心框架已完成事件系统、属性存储、Buff 与修饰符管线以及基础 AI 规划器等模块。这些模块提供了存储数值、修改属性、驱动 Buff 逻辑和基本决策的能力，但尚未引入姿态压力、护甲完整度等"压力类属性"，也未将属性变化用于 AI 决策。

### 目标

构建一种既符合玩家直觉又易于扩展的属性系统，将它与自动战斗 AI、环境互动、跨作品能力映射和复盘工具结合，形成一个完整可玩闭环。最终让"电子斗蛐蛐"与艾尔登法环式 ARPG 交互融合，同时能够解构各种游戏或非游戏作品的能力并映射到统一机制。

---

## 二、设计原则

### 原则一：玩家语义优先

每个属性都应该反映可感知的战斗状态。例如"姿态压力"对应站不稳或将被打崩；"护甲完整度"对应外壳是否即将破裂。属性不是抽象数字，而是玩家直觉上能理解的状态。

### 原则二：机制闭环

属性不仅是后台数值，还要定义积累、恢复、触发、反馈全过程，形成可预期的闭环。例如姿态压力超过阈值后触发破韧事件，进入硬直或处决窗口。

### 原则三：事件驱动

属性变化通过事件总线广播，UI、AI、能力系统分别订阅，不直接耦合。这样既满足确定性，又方便扩展和调试。

### 原则四：扩展友好

采用配置化和模块化设计，支持未来添加新的属性类型、环境交互、能力模板以及不同体型（蛐蛐、人物、机械）的差异化实现。

## 三、属性系统重构

### 3.1 新增属性类型

为了让属性更好地驱动机制，本设计按属性功能划分出以下几类：

| 类型 | 功能描述 | 示范属性 |
|------|---------|---------|
| 压力属性 (Pressure) | 可积累、可衰减，跨阈值时产生状态段变化并触发事件 | 姿态压力、防御压力、疲劳压力、外骨壳完整度 |
| 资源属性 (Resource) | 可消耗、可回复，用于技能释放或行为限制 | 体力、怒气、专注、节奏值 |
| 状态属性 (State) | 表示持续或瞬时的状态标记 | 湿润、热量、电荷、腐蚀 |
| 关系属性 (Relation) | 描述与他人或环境的互动关系 | 威胁、警觉、恐惧压力 |

### 3.2 压力属性模型

#### 结构定义

```csharp
class PressureAttribute
{
    float current;                 // 当前压力
    float recoveryRate;            // 每秒恢复量
    float maxValue;                // 最大值，可作为压崩上限
    PressureBand band;             // 当前状态段
    Thresholds thresholds;         // 临界阈值 (Stable, Pressed, Cracked, Critical)
    Hysteresis hysteresis;         // 避免抖动的迟滞区间
}
```

#### 累积与恢复

每种压力属性定义自己的累积来源（受击、格挡、环境影响等）和恢复规则（脱战恢复、稳固动作恢复、技能清除等）。

#### 状态段 (PressureBand)

分为 Stable、Pressed、Cracked、Critical、Broken 等几段，用于驱动 UI 表现和 AI 决策。离开临界段需要低于回撤阈值，避免频繁闪烁。

#### 事件

- 当状态段发生变化时广播 `PressureBandChangedEvent`
- 当压力超过 maxValue 或阈值线时广播破裂事件（例如 `PostureBreakEvent`）

### 3.3 核心压力属性示例

#### 姿态压力 (PosturePressure)

| 维度 | 说明 |
|------|------|
| 累积来源 | 被命中、格挡震退、爆炸冲击、推撞等 |
| 恢复方式 | 未受击时间、脱战、稳固姿态动作 |
| 阈值段 | Stable → Pressed → Cracked → Critical → Broken |
| 破裂效果 | 破韧，强制打断动作并进入硬直 |

**支撑机制**：ActionSupport 定义当前动作的抗断线（基础值 × 动作系数 × 最低保底）。破韧判断为 PosturePressure >= currentBreakLine。支撑线只表现为护框，不改变压力值本身。

#### 防御压力 (GuardPressure)

| 维度 | 说明 |
|------|------|
| 累积来源 | 格挡攻击、连续承受重击、盾牌震退 |
| 恢复方式 | 停止格挡、成功格挡反击、短暂后退 |
| 破裂效果 | 触发 GuardBreakEvent，角色暂时无法格挡并进入破防硬直 |

#### 外骨壳完整度 (ArmorIntegrity)

| 维度 | 说明 |
|------|------|
| 特性 | 类似耐久度，不通过恢复而只能修复或升级 |
| 效果 | 完整度降低会减少伤害吸收并降低破韧阈值 |
| 破裂事件 | 触发 ArmorBreakEvent，角色暴露弱点并更容易被打崩 |

#### 疲劳压力 (FatiguePressure)

| 维度 | 说明 |
|------|------|
| 累积来源 | 连续出招、连续闪避、重武器挥动、超频振动等 |
| 效果 | 压力越高，动作承诺和后摇时间越长，部分技能不可用 |
| 临界效果 | 进入疲劳状态，需要休息或消耗资源解除 |

## 四、AI 机制整合

### 压力传感器

在 AI Planner 中新增传感器接口，例如 `ReadPostureBand(entity)`、`ReadArmorIntegrity(entity)`，使决策模块感知己方和敌方的当前状态段。

### 效用驱动决策

在 Utility AI 或行为树的权重计算中注入姿态与防御信息：

- 当敌方姿态处于 Critical 时，增加使用高削韧技能的权重
- 当自身 GuardPressure 过高时，优先切换防御姿态或后撤

### 行为触发规则

利用事件总线订阅 `PressureBandChangedEvent`，在状态段切换时触发行为，例如 `OnPostureCritical` 自动使用回避或招架技能。

### 学习与适应

在关系属性中维护 Threat、Confidence 等指标，AI 根据连续战斗结果调整攻击或防守策略。

## 五、环境与交互层

### 牵引力 (Traction)

定义不同地面或物体表面的抓地力，影响移动速度、冲锋距离和击退距离。冰面降低牵引力，泥地增加阻力，草丛隐藏行动。

### 环境状态属性

湿润 (Wetness)、热量 (Heat)、电荷 (Charge)、腐蚀 (Corrosion) 等环境属性既可以作用于角色，也可以是区域状态，用于驱动元素交互：

- 湿润提升电荷累积，降低火焰持续时间
- 热量压力过高导致过热爆炸或燃烧
- 电荷压力过高导致麻痹或连锁电击
- 腐蚀压力作用于护甲完整度，触发破壳

### 区域压力 (ZonePressure)

用于表示战场某个区域的危险度或效果（火焰区、毒雾区、音波共振区），AI 可以利用或避开这些区域。

## 六、能力映射层

### 能力基元

将外部作品的技能拆解成若干基本效果：施加姿态压力、增加外骨壳完整度、施加毒素压力、改变 Traction 等。

### 能力模板

定义能力的触发条件、持续时间、目标筛选、影响基元及数值参数。以配置（JSON 或 ScriptableObject）形式存储，支持热更新和 Mod 加载。

### 加载器

解析能力模板生成运行时效果对象，将基元效果转化为属性变化或事件，并与 Buff/Modifier 管线集成。

---

## 七、UI 与表现层

### 压力条与护框

为每个重要压力属性设计独立的条形 UI：

- 姿态压力条：显示当前压力和分段
- 护框条：显示当前动作支撑线，突破护框时播放裂纹爆裂动画
- 防御压力条：格挡时显示，破防时条爆裂并伴随盾碎视觉特效

### 状态反馈

使用多模态效果（光效、材质变化、慢动作、音效）展示压力段变化与破裂事件，确保玩家在不读数值的情况下也能直观感知。

### 调试视图

开发者模式下可显示所有 Pressure 属性的实时数值与事件流，便于调试和性能分析。

---

## 八、复盘与诊断

### 压力变化日志

在 CombatReplay 或 TimelineScrubber 中记录每次压力增减、阈值段变化、破韧/破防/破壳事件，并与动作时间轴和动画轨迹对齐，供开发者和玩家查看战斗过程。

### 分析工具

提供可导出的 JSON 或图形化界面，用于分析战斗中各属性值的走势、AI 决策依据和能力效果触发情况，支持构筑验证和策略调整。
